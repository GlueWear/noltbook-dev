Tests the formal Noltbook Developer API v1 over Eyre: reads via api.scry('noltbook','/api/notes') and writes via api.poke('noltbook','noltbook-api',…) — a stable surface, no access to noltbook internals. (Phase 1 used the raw noltbook-action mark; this is the formal veneer.) Selected note: none

2 · CREATE / FIND "api-test"

scry /notes, match name api-test; else poke create-note and re-scry until it appears.

2.5 · ACTOR IDENTITY (optional, app-scoped)

Sent as top-level actor {id,name,kind} on post-message / post-app-ref (and the DM/gossip tests below). Honored only when app is valid; backend stamps host=our + desk=app.desk. Regular notes & DMs only — cover/gossip/ars-rumors omit actor in v1. Display/attribution only.

Phase A: an actor is honored only if its app desk is granted and the actor is active (see Actor Control). A denied actor rejects the post (watch the result line) — it is never silently posted as the host. First post after granting auto-registers the actor (TOFU).

2.6 · ACTOR CONTROL (host governance · Phase A)

Host grants which local app desk may attribute actors, and each actor's lifecycle. Cooperative, not authenticated: Gall does not expose the calling agent, so desk is app-supplied — this is revocation/audit, not isolation from a malicious local agent.

7 · MEMBERSHIP / ADMIN (mutations)

Poke noltbook-api membership/admin actions on the selected note. Each sends a requestId; watch the result line/log above. Routes through Noltbook's real handlers (host/admin checks enforced).

8 · NOTE CONFIG (mutations)

Poke set-note-config on the selected note. Only filled fields change; leave a field blank to keep it. Routes through Noltbook's rename / set-note-meta / set-headline handlers (creator-only).

8b · APP METADATA (set-note-app)

Durable note-level association with a Noltbook-capable app (not a note type). Creator/admin only; notebook/group/gossip notes. Reads appear as app in /api/notes and /api/notes/<id>/meta. app:null clears.

8c · ACTIVE (set-note-active)

Developer/API-only note "live" status (separate from calls). Requires top-level app attribution (harness sends APP). Creator/host only; notebook/group/gossip notes. TTL heartbeat in seconds (default 120, cap 600); label caps at 32 (empty → "live"); count caps at 999. Reads appear as active in /api/notes and /api/notes/<id>/meta.

9 · ARTIFACT CREATE

Poke create-artifact (code/app only — no raw file upload in 12A) into the selected note, with APP attribution. Result shows artifactId/eid; the new artifact appears in Read Recent's artifact rows.

10 · SEARCH MESSAGES

Poke search-messages (requestId required) — result arrives on /api/results as api-search-result. Click a hit to open its note. Skips cover/ars-rumors/blocked; cap 50.

11 · DM FIND / CREATE

Poke find-or-create-dm for a target ship. On found/dm-created the DM note is selected + read (idempotent — run twice to get the same noteId). requestId required.

12 · GOSSIP NOTE FIND / CREATE

Poke find-or-create-gossip-note (idempotent by exact name + creator=our + type=gossip). On found/gossip-created the note is selected + read. Creates **user-gossip** notes only.

13 · FORKS

fork-note the selected note (group notes only; member/removed). /api/fork-invites lists pending invites with accept/decline. accept is **async** — it requests a remote fetch + installs later.

14 · CALLS

Noltbook call state only — no media/WebRTC. Start/join/leave operate on the selected note's call. Reads: /api/calls, /api/notes/<id>/calls.
